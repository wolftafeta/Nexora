Why the name

What is Nexora?

We didn't pick a random name. Nexora was built around a concept the idea that every great Discord server needs a single, reliable core to hold everything together.

From tickets to moderation to giveaways, Nexora is that core. The central point where every system connects.

From Nexus (Latin: connection, the point where things meet) + ora (evoking structure, precision, and order). Together: the connection point that brings order to your server.

Quick Setup

Getting Nexora running on your server takes less than five minutes. Follow these steps to get the core systems online.

1
Invite Nexora to your server using the invite link and grant it Administrator permissions for full functionality.
2
Run /setup in the channel you want to use as your settings channel. Nexora will post interactive panels there.
3
Use the Tickets panel to create your first ticket panel. Click "Create Panel" and follow the prompts.
4
Enable Moderation modules you want active bad word filter, anti spam, etc. from the Moderation panel.
5
Set your log channels from the Logging panel. Each log type can go to its own channel.
Tip: Run /setup wizard for an interactive guided setup that walks you through every step with buttons and menus.
⚙️ Core

Settings Dashboard

The settings dashboard is a set of interactive embeds posted inside your settings channel. Each embed covers a different system and updates live as you make changes.

The main panel shows live server statistics including active tickets, total bans, mutes, warnings, support staff count, and active punishments.

  • All configuration happens directly in Discord no external website needed
  • Panels update in real-time when settings are changed
  • Separate panels for Tickets, Moderation, Logging, Giveaways, and Bot settings
  • Buttons, dropdowns, and modals for all interactions
  • Live blacklist management built into the dashboard
🎫 Tickets

Ticket System

Nexora's ticket system supports both single and multi-panel setups, with full customization over every ticket type.

Panel types: A single panel has one ticket type and one button. A multi-panel has multiple ticket types selectable via buttons or a dropdown menu.

Ticket naming: Use tokens to format channel names automatically {username}, {number}, and {type} are available.

Status icons update the channel name automatically as the ticket progresses:

  • 🎟️ ticket created, not yet seen by staff
  • ⌛ staff has sent their first message
  • ✅ a close request has been started
Note: All status icons are fully customizable from the Settings Dashboard under Ticket Settings.

Available ticket commands: /close · /closerequest · /add · /remove · /rename · /priority · /transcript · /delete · /move · /panel · /ticketinfo

🛡️ Moderation

Moderation

Nexora includes a fully modular moderation system. Each module can be enabled, disabled, and configured independently.

Bad word filter: Add words to the filter list. Wrap a word in ** to also catch bypass attempts like leetspeak, repeated characters, and unicode lookalikes.

Punishment options for each module: delete message · warn · timeout · kick · ban

Each module supports: ignored roles, ignored channels, ignored categories, custom punishment duration, and channel-specific overrides.

  • Bad Word Filter with ** similar detection
  • Anti Spam configurable message threshold and interval
  • Anti Mention Spam threshold per message
  • Anti Advertisement Discord invites and external links
  • Link Protection allowlist specific domains
📋 Applications

Application System

The application system lets you create professional application panels similar to ticket panels, with full control over questions and the review process.

Submission methods: Applications can be submitted via Discord modal forms (up to 5 questions) or via DM, where the bot asks questions one by one (unlimited questions).

Question types: short answer · paragraph · multiple choice · required · optional each with custom placeholders.

Review workflow: Accepted applications turn the embed green; denied turn it red. The reviewer's information and timestamp are automatically added to the embed.

Interview tickets: Enable interview tickets to automatically create a private ticket channel when an interviewer clicks the Interview button on an application.
📊 Logging

Logging

Every log type has its own channel setting and can be toggled on or off independently. You can also combine related logs into a single channel using combined sets.

  • Message delete and edit logs with full content, author, and attachments
  • Role logs showing who gave/removed a role and who made the change
  • Member join/leave, nickname, username, and avatar change logs
  • Voice logs: join, leave, move, server mute/deafen
  • Channel and server update logs
  • Ticket and application logs
  • Automod trigger logs with full context

Each module also supports ignored channels, ignored categories, ignored roles, and custom embed colors.

🎁 Giveaways

Giveaway System

Create giveaways quickly with the quick setup, or fully customize every aspect of the embed and requirements.

Quick setup: Run /gcreate and fill in the prize, duration, and winner count. Nexora generates a clean embed automatically.

Requirements: Set minimum account age, minimum server join time, and required roles to limit who can enter.

  • /gcreate create a new giveaway
  • /gend [id] end a giveaway early
  • /greroll [id] select a new winner
  • /gpause [id] pause an active giveaway
  • /glist view all active giveaways
📁 Transcripts

Transcripts

When a ticket is closed, Nexora automatically generates an HTML transcript that looks exactly like a Discord chat archive.

The transcript page includes profile pictures, usernames, timestamps, rendered embeds, and attachment links all with Discord's visual style.

A unique link is generated for each transcript and included in the transcript embed posted to your log channel. The embed also shows the ticket creator, who closed it, the panel name, open time, close time, and total duration.

Note: Transcripts are hosted on your own Railway/Render deployment alongside the bot, so no external service is needed.
🎨 Customization

Bot Customization

Every visual aspect of Nexora is configurable per server from the Bot Settings panel. The bot starts with a neutral grey theme by default.

  • Embed colors global default and per-panel overrides
  • Embed icons, thumbnails, and images
  • Footer text and footer icons
  • Button styles (Primary, Secondary, Success, Danger)
  • Custom embed titles and descriptions
  • Command prefix for prefix commands
  • Custom branding name and logo
🚫 Blacklists

Blacklist System

Nexora maintains separate blacklists for each system. A user blacklisted from tickets is not automatically blacklisted from giveaways unless you configure it that way.

  • Ticket blacklist prevents opening tickets
  • Giveaway blacklist prevents entering giveaways
  • Application blacklist prevents submitting applications
  • Moderation blacklist ignored by automod
  • Command blacklist blocks specific command usage

All blacklists are manageable directly from the Settings Dashboard. You can add users, set reasons, view history, and search by username or ID.
